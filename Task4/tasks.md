# Крупные задачи MVP (6 месяцев) + целевое состояние (12 месяцев)

## Сайт
- Добавить страницу депозитов: список продуктов и ставок (чтение из сервиса ставок).
- Добавить форму заявки (ФИО + телефон) + согласия/валидация.
- Интеграция с сервисом заявок (создание лида/заявки), обработка ошибок/повторов.
- Логи/метрики для конверсии и ошибок.

## Интернет-банк
- Экран депозитов: список продуктов/ставок + персональные ставки.
- Подача заявки: счёт + сумма + подтверждение СМС (использовать текущий механизм).
- Интеграция с сервисом заявок и сервисом ставок по HTTPS/JSON.
- Оптимизация справочников/кэширование (снять >1 сек задержки по справочным данным).

## Сервис ставок (RPS)
- Модель данных ставок/версий, аудит изменений.
- API: получить ставки (публичные), получить персональные (для ИБ), получить метаданные версии.
- Интерфейс/регламент загрузки ставок из XLS (переходный этап) + роли “депозиты/кредиты”.
- Кэширование и публикация новой версии.

## Сервис заявок (DAS)
- API: создать заявку (сайт/ИБ), статусы, получить очередь заявок бэк-офису.
- Хранилище заявок, идемпотентность, корреляция request-id.
- Интеграция с кол-центром (создание лида/уведомления) — вариант зависит от возможностей системы КЦ.

## Бэк-офис (Workbench)
- UI для очереди заявок, статусы, комментарии, история действий.
- Переход в АБС для подтверждения/открытия депозита (MVP) + фиксация статусов в DAS.

## АБС + адаптер
- Контракт интеграции (что можно автоматизировать в MVP, что оставить вручную).
- Адаптер доступа (контролируемые вызовы/лимиты).
- (Опционально) автоматизация передачи статуса “депозит открыт” обратно в DAS.

## Кол-центр (внутренний)
- Отображение лидов с сайта, результат звонка.
- Доступ к актуальным ставкам из RPS (внутренний API-клиент + кэш).
- Скрипт консультации с указанием версии ставок.

## Партнёрский кол-центр
- Регламент приёма файла ставок.
- Импорт файла (ручной/полуавтомат).
- Контроль версии/времени обновления (процесс).

## Инфраструктура/безопасность/эксплуатация
- TLS, секреты, доступы сервисов.
- Логи/метрики/алерты (ошибки, задержки отправки файла партнёру).
- Развертывание в 2 ЦОД (failover), резервирование БД.
